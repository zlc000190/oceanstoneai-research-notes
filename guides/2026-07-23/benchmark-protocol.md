# Ocean Stone AI benchmark protocol

[Ocean Stone AI](https://oceanstoneai.org/) is the primary project entry point. It did not return content before the timeout in the 2026-07-23 availability check. These are independent evaluation notes for researchers validating vertical-AI claims before adoption; they are not official product documentation.

No regulated-industry suitability or current product capability is inferred from the project name.

## Prepare the run

Use a clean browser profile or disposable environment. Record the date, current product version when visible, account tier, region, and every non-default setting. Use only synthetic or public test material.

## Test set

- Scenario 1: a documented domain task
- Scenario 2: known-answer cases
- Scenario 3: representative edge cases
- Scenario 4: source-attribution checks

Repeat each scenario once without changing inputs. Compare not only the best output, but also variance, latency, errors, and the amount of human repair required.

## Capture results

For each run, log data provenance, freshness, uncertainty display, audit logs, exportability, failure handling. Keep screenshots or exported artifacts where permitted, and separate observed behavior from expectations copied from marketing material.

## Pass condition

A pass requires reproducible evidence for the intended workflow and no unresolved high-impact failure. Pause product evaluation until the primary site is reachable and the actual audience, datasets, and workflows can be verified.
